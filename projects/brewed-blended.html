A Café Ordering System

the challenge

What problem did we solve?

Manual café ordering and paper ticketing systems slow down customer queues, introduce pricing mistakes, make it confusing to modify carts, and leave managers without a clean way to track inventory or run live transaction analysis. Small cafés run on thin margins — inefficiency is expensive.

what we built

How did we solve it?

We engineered a normalized relational database infrastructure linking users, orders, and products with full referential integrity. The integrated Admin Panel features a built-in graphical SQL Query Builder, allowing café managers to perform immediate aggregate calculations — top-spending customers, best-selling items — without needing to write raw SQL.

what i learned

What did this teach me?

Brewed&Blended was my first real experience building a desktop GUI from scratch using Java Swing, and it humbled me quickly. Making something look acceptable in a browser is one thing — laying out a GUI manually, handling component sizing, managing state across panels — is a completely different problem. I developed a much stronger mental model of how event-driven programming works: the user clicks something, an event fires, something updates, the UI redraws. Getting that flow right across a complex ordering system with a cart, menu, and admin panel running in the same application taught me to think about user interactions as state transitions rather than just button clicks. I also learned that building for non-technical users means your error messages and UI feedback need to be bulletproof — they won't read documentation, they'll just click things and expect it to work.
