Bank Operations System

the challenge

What problem did we solve?

Managing everyday personal banking or tracking transactions using paper records or scattered files is slow, prone to math mistakes, and lacks a simple way to instantly verify whether an account has enough funds before a withdrawal goes through.

what we built

How did we solve it?

We created a unified Python command-line system that models real-world account structures using object-oriented logic. It handles deposits and withdrawals safely by instantly checking available balances, enforces strict validation to block invalid inputs, and keeps customer details updated in a central data structure.

what i learned

What did this teach me?

BTMS was where OOP stopped being a concept I understood in theory and became something I actually used to solve a real design problem. Before this, I'd write classes because the assignment said to. Here, the structure genuinely mattered — modeling an account as an object with its own state and behavior made the code dramatically easier to reason about and extend. I also had to think hard about validation: what happens when someone enters a negative number? What about withdrawing more than the balance? Handling edge cases isn't glamorous, but getting them wrong in a banking context means the system is fundamentally untrustworthy. That discipline of thinking through failure modes before they happen has stayed with me.
